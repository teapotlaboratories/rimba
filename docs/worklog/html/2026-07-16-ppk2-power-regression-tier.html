A PPK2 power-save regression tier (tp)

A new regression tier that meters board2's power-save ladder off the Nordic PPK2 and gates a gross current regression — the kind the fw-1.17.9 ~2× PS regression was — while always recording the raw mA. Built end-to-end, calibrated on-rig, and validated: a fresh 1.17.8 run scores 4 PASS, and a synthetic sweep proves a 2× regression FAILs (after that sweep caught a scoring bug).

Bottom line

make test-tp / python tools/regtest/run.py tp --ap esp|linux owns the PPK2, power-cycles board2, brings up an AP (ESP SoftAP or Linux hostapd_s1g), flashes the C6-triggered regtest-power ladder, samples the current stream, segments it by the firmware's REGTEST| phase markers, scores Dyn-PS + WNM against calibrated per-AP bands, and radio-silences. It is a new tier, not a T2 test — a power verdict comes from a host-side PPK2 stream the firmware can't see.

Owner decisions

  • Both AP paths — ESP32 SoftAP (regtest-apsta-ap, fully automated) and Linux hostapd_s1g on chronite (the authoritative reference). One DUT serves both: the config hostapd-rimba.conf matches the ESP AP and the DUT exactly (ssid=rimba-ping / SAE / dtim_period=1 / PMF / ch27).
  • Wide gross-multiple gate — FAIL only at ~1.8–2× the calibrated baseline; INCONCLUSIVE in the grey zone; raw mA always recorded; drawing below the band is never a FAIL. Honors the repo doctrine that doze depth is a benchmark, not a pass/fail.
  • Keep the C6 trigger — reuse the existing C6 (GPIO20 → board2 D5) that pulses every 30 s; no self-trigger firmware. The host samples the PPK2 and segments off the console.
  • Calibrate now, harden later — lock the as-is (RX-overload-offset) bands so the gate is live now; file the overload reduction as v2.

The tp pipeline

The ladder runs four 18 s tiers; the firmware emits a REGTEST|INFO|phase=N <key> start marker at each tier entry (the host's segmentation timestamps) plus structural STEP facts (associated, twt-installed, wnm-accepted) — and, deliberately, no REGTEST|RESULT, because the firmware can't see the current.

#tierAPIrole in the gate
1No-PSset_power_save_mode(DISABLED)validity anchor (~65 mA → associated + not overloaded)
2Dyn-PSset_power_save_mode(ENABLED)scored

The finding: the bench reads ~2× the reference

Every doze tier reads ~1.6–2× the documented reference on both APs, but No-PS validity passes (STA associated, ~63 mA), so the measurement is real. It is a close-bench RX-overload signature: the DUT caps its own TX to 1 dBm, but the AP blasts it at full power from cm away, so the radio keeps waking on retries (an elevated p10 floor + spike-driven p90).

The synthetic sweep caught a scoring bug

🐛 tier-separation guard masked the very regression it should catch

The first cut gated scored tiers on a validity check dyn-ps < 0.7 × No-PS (meant to detect "PS never engaged"). But a real regression pushes doze current up toward No-PS — so that check failed exactly on a regression, scoring a ~2× regression INCONCLUSIVE instead of FAIL, defeating the gate's whole purpose.

Fix: validity is now just associated + No-PS in window; a high scored median is what fail_min is for. Rather than flash 1.17.9 (out of scope for "calibrate now"), the gate logic was validated with a synthetic sweep through _score, which is what surfaced this.

Disposition + v2

The tp tier is complete and live (all in the already-uncommitted regtest harness). T0 stays green (the drift check passes, regtest-power builds). A teardown bug was found and fixed along the way: the runner used to re-spawn ppk2_hold from its finally, which raced the just-closed PPK2 handle and crashed it; now it drains the stream and leaves board2 latched-powered instead.

v2 — reduce the RX-overload offset

Cap the AP's TX too — the ESP AP via mmwlan_override_max_tx_power, the Linux AP via the morse tx_max_power_mbm module param on chronite — for reference-like numbers (Dyn-PS ~9–15, WNM ~5), then a tighter recalibration that catches a smaller regression. Optional: the D5→PPK2-logic-input wire for inline phase self-labeling; a HOST_LIGHT_SLEEP build as a stronger secondary gate.

Bench radio-silent after (board0/board2 on rimba-hello, chronite wlan1 down, board2 held by tools/ppk2_hold.py).
